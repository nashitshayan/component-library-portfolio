git, github, VSCode, and hosting

June 07, 2021

In the lesson 2 of levelZero, I learned :

understanding how git works

Git, in the most layman terms, is basically a way to keep track of changes in the code. It tells us what changes were made, who made those changes, and when were these changes made. This ensures that if ever we make a mistake, we can revert the changes and fix the problem.

understanding how github works

The name kinda gives it away. It is used for collaboration. Essentially, the code is put up on the cloud, online. People can consume it, collaborate on it, etc. I can work on a project with a person who’s on the other side of the world. We can work on the same code, see each other’s changes, etc. This becomes very useful in big organisations, because things can’t be kept on the local machine of the developer, there’s risk of loss of data. Having the code online ensures security.

writing good commit messages

  • Feat- a new feature
  • Fix- a bug fix
  • Docs- changes in the documentation
  • Refactor- changes in the code that don’t affect the functionality or the output. No feature is added nor any bug is fixed.
  • Style- everything related to styling
  • Test: everything related to testing
  • Chore- update build tasks, package manager configs, etc

understanding how internet works

Whenever we enter a url, the browser sends a request to a server (domain name server), that gives back the ip address of the server where our website is.

Then our browser goes to that ip and fetches the website. It downloads the html file and that’s what we see.

using netlify to host websites

netlify is a free hosting service that allows you to host your website.Our assignment was to host our new portfolio website that we'll develop in the coming lessons

using VSCode

  • show/hide terminal : ctrl + `
  • show/hide sidebar : ctrl + B
  • open file : ctrl + P, search for file name
  • command pallete : ctrl + shift + P
  • jump to line number : ctrl + P, then :linenumber
  • jump to symbol : ctrl + P, then @
  • toggle wrap : alt + Z
  • save file ctrl + S
  • there are many more, can look up them at "keyboard shortcuts"
